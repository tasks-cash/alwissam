# API Inventory

Source-verified **2026-07-13** from all `src/app/api/**/route.ts` files (**30**) plus `/patient/qr/[token]` (**31 handlers**). Rows are **one per HTTP method**.

## Cross-cutting security & runtime facts

- **CSRF:** Mutating authenticated routes compare header `x-csrf-token` to `session.csrfToken` (via `getCurrentUser().csrfToken`). Mismatch → `403` Arabic error `رمز الحماية غير صالح`.
- **Auth:** Almost all protected handlers call `getCurrentUser()` and check `user.role.code` against a role array. **No handler calls `requirePermission()`.**
- **Sessions:** Cookie name `alwisam_session` (`SESSION_COOKIE` in `src/lib/auth/session.ts`).
- **Passwords:** `hashPassword` / `verifyPassword` use **bcryptjs** (12 rounds) in `src/lib/auth/password.ts`.
- **Redis:** `publishEvent` on exam/arrive/waiting-room handlers; appointment service also publishes `clinic:appointments` / `clinic:waiting-room` for walk-in, direct, check-in flows.
- **Notable gaps:** `/api/secretary/schedule-appointment` **always returns 403**; `/api/files/upload` allows **any authenticated user** with **no patient ownership check**; `/api/realtime/stream` is **SSE for any authenticated user** and **polls DB** every 5s.
- **Status:** all `legacy_active`. **Tests:** all `none`. **Target NestJS Endpoint:** same path preserved under Nest.
